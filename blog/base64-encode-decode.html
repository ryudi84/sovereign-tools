Base64 Encode & Decode: When and How to Use It

Published Feb 22, 2026 ¬∑ 5 min read

Base64 is one of those things every developer uses but few truly understand. You've seen it in data URIs, email attachments, API payloads, and JWT tokens. This guide explains what Base64 actually is, how it works under the hood, and when you should (and shouldn't) use it.

What Is Base64?

Base64 is a binary-to-text encoding scheme that represents binary data using 64 printable ASCII characters. It was designed to safely transmit binary data through systems that only handle text ‚Äî like email (SMTP) or JSON payloads.

The 64 characters used are:

A-Z  (26)  ‚Üí  values 0-25
a-z  (26)  ‚Üí  values 26-51
0-9  (10)  ‚Üí  values 52-61
+    (1)   ‚Üí  value 62
/    (1)   ‚Üí  value 63
=          ‚Üí  padding

üí° Base64 is encoding, not encryption. It provides zero security. Anyone can decode it. Never use Base64 to "hide" sensitive data.

How Base64 Works (Step by Step)

Here's how the string "Hi" gets encoded:

  1. Convert to bytes: H = 72, i = 105 ‚Üí binary: 01001000 01101001
  2. Group into 6-bit chunks: 010010 000110 1001 (pad to 24 bits ‚Üí 010010 000110 100100)
  3. Map to Base64 alphabet: 18=S, 6=G, 36=k
  4. Add padding: Since 2 bytes need 1 pad char ‚Üí SGk=

Every 3 bytes of input become 4 characters of Base64 output. That's why Base64 increases size by ~33%.

Base64 vs Base64URL

Standard Base64 uses + and /, which are special characters in URLs. Base64URL replaces them:

EncodingChar 62Char 63Padding
Base64+/=
Base64URL-_Optional/omitted

Base64URL is used in JWT tokens, OAuth, and anywhere tokens appear in URLs.

When to Use Base64

‚úÖ Good Use Cases

  • Data URIs ‚Äî Embed small images directly in HTML/CSS: data:image/png;base64,iVBOR...
  • Email attachments ‚Äî MIME encoding uses Base64 to send binary files through text-only SMTP
  • JSON payloads ‚Äî When you need to include binary data in JSON (which only supports text)
  • Basic Authentication ‚Äî HTTP Basic Auth encodes username:password in Base64
  • Storing binary in text fields ‚Äî Database text columns, environment variables, config files

‚ùå Bad Use Cases

  • Security/encryption ‚Äî Base64 is trivially reversible. Use actual encryption (AES, RSA).
  • Large files ‚Äî The 33% size overhead adds up. Use proper binary transfer instead.
  • Obfuscation ‚Äî Anyone can decode Base64 in seconds. It's not hiding anything.

Base64 in Every Language

JavaScript (Browser)

// Encode
const encoded = btoa('Hello, World!');
// ‚Üí "SGVsbG8sIFdvcmxkIQ=="

// Decode
const decoded = atob('SGVsbG8sIFdvcmxkIQ==');
// ‚Üí "Hello, World!"

// For Unicode strings:
const encoded = btoa(unescape(encodeURIComponent('H√©llo')));
const decoded = decodeURIComponent(escape(atob(encoded)));

Node.js

// Encode
Buffer.from('Hello, World!').toString('base64');

// Decode
Buffer.from('SGVsbG8sIFdvcmxkIQ==', 'base64').toString('utf8');

Python

import base64

# Encode
base64.b64encode(b'Hello, World!').decode()

# Decode
base64.b64decode('SGVsbG8sIFdvcmxkIQ==').decode()

Command Line

# Linux/Mac
echo -n 'Hello' | base64        # encode
echo 'SGVsbG8=' | base64 -d     # decode

# Windows PowerShell
[Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes('Hello'))
[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String('SGVsbG8='))

Common Pitfalls

  • Unicode issues in browsers ‚Äî btoa() only handles Latin1. For UTF-8, you need the encodeURIComponent trick shown above, or use TextEncoder.
  • Newlines in encoded output ‚Äî Some implementations add line breaks every 76 characters (per MIME spec). Strip them if you don't need them: encoded.replace(/\n/g, '')
  • Padding confusion ‚Äî The = padding isn't always required. Base64URL often omits it. But standard Base64 decoders may fail without it.
  • Double-encoding ‚Äî Base64-encoding an already-encoded string creates a mess. Always decode first, then re-encode if needed.
