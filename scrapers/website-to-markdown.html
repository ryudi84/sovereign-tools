from apify_client import ApifyClient

client = ApifyClient("YOUR_API_TOKEN")

# Feed chunks directly into your vector DB

Frequently Asked Questions

Any publicly accessible website. Documentation sites, blogs, news sites, wikis, and knowledge bases work best. Sites with heavy JavaScript rendering may require the browser-based crawling mode.

Set a chunk size (e.g., 1,500 characters) and overlap (e.g., 200 characters). The tool splits content into chunks of that size with the specified overlap between consecutive chunks, preserving context for embedding models.

The output works with any vector database: Pinecone, Weaviate, Chroma, Qdrant, Milvus, pgvector, and more. The chunked text output can be directly embedded using OpenAI, Cohere, or any embedding model.

The crawler can be configured to follow or ignore robots.txt. We recommend respecting it. You can also set rate limits (requests per minute) to be gentle on target servers.
